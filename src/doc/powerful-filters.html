Information préliminaires

Vocabulaire

  • filtre allumé : le filtre est dans l'état allumé si et seulement si au moins un de ses groupes est allumé
  • groupe allumé : le groupe est dans l'état allumé si et seulement si au moins un de ses inputs est allumé
  • input allumé : l'input est dans l'état allumé si et seulement s'il est utilisé/coché/rempli (attention, si l'input est inversé, alors il est dans l'état allumé si et seulement si il n'est pas utilisé/coché/rempli)

Comment ajouter un attribut

Les attributs sont la base de la solution de filtre de Codelius, voici comment ajouter un attribut dans Webflow :

cd-filter-list *

❓

Permet d'indiquer où se trouvent les éléments qui devront être filtrés
Prendra en valeur, le nom du filtre que vous voulez créer

📍

A placer sur l'élément contenant les éléments à filtrer

🥳

A noter que la solution de filtre de Codelius permet de filtrer n'importe quelle liste d'élément, pas forcément des Collection List du CMS !

Si vous utiliser un CMS et que la pagination est activée, alors tous les items sont téléchargés et insérés au fur et à mesure afin de garder un chargement rapide et de respecter les limitations de Webflow

cd-filter-flag *

❓

Permet de rediriger une action sur un element relatif à la cible (uniquement si précisé compatible)

📍

A placer sur l'élément contenant les éléments à filtrer

🥳

A noter que la solution de filtre de Codelius permet de filtrer n'importe quelle liste d'élément, pas forcément des Collection List du CMS !

Si vous utiliser un CMS et que la pagination est activée, alors tous les items sont téléchargés et insérés au fur et à mesure afin de garder un chargement rapide et de respecter les limitations de Webflow

cd-filter-adder *

Ce nombre a un maximum précisé par la doc
Il fonctionne avec des opérateurs
A chaque fois que le adder est appelé, il joue un opérateur, la prochaine fois il jouera l'opérateur suivant
Une fois arrivé au dernier opérateur, il boucle sur ce dernier

➡️

all

Amène le nombre a son maximum

➡️

gX

Amène le adder à l'opérateur X (en partant de 0 : 0, 1, 2, 3 si 4 opérateurs) cela continue tant qu'on est pas tombé sur un opérateur autre que gX (maximum 10 goto d'affilé)

➡️

=X

Amène le nombre à X

➡️

xX

Multiplie le nombre par X

➡️

+X

Ajouter X au nombre

Exemple avec les opérateurs ="+10, +3, x2, g1"
Il va ajouter 10 la première fois, au second déclenchement il ajoute 3, puis multiplie par 2, puis recommence à l'étape 2
Si on part de 0 et qu'on a un maximum de 100: 0 -> 10 -> 13 -> 26 -> 29 -> 58 -> 61 -> 100 -> 100 -> 100 etc...

Exemple avec l'opérateur ="=100, x2"
Il va aller à 100 la première fois, puis multipliera par 2 indéfiniment
Si on part de 5 avec un maximum de 1000: 5 -> 100 -> 200 -> 400 -> 800 -> 1000 -> 1000 -> 1000 -> etc...

cd-filter-paginate *

Défini la longueur d'une page, la longueur d'une page permet de limiter le nombre d'élément affiché même si plus d'items ont passés les filtres

Prend comme valeur un entier, strictement supérieur à 0
Aucune limite par défaut

📍

A placer au même endroit que la cd-filter-list

Configuration minimale obligatoire

Liste à filtrer

Je veux que cette liste d'éléments soit ciblée par mon filtre.
Ajouter l'attribut
name
cd-filter-list
value
nom_du_filtre
Description

Cet attribut se place sur l'élément qui contient tous les éléments qu'on souhaite filtrer. Généralement un CMS mais vous pouvez aussi le placer sur n'importer quelle div.

Le nom_du_filtre sera utilisé dans plusieurs autres attributs pour lier les composants à la liste à filtrer.

Contexte

Je veux que cet élément soit lié à ma liste à filtrer.
Ajouter l'attribut
name
cd-filter-ctx
value
nom_du_filtre
Description

Cet attribut se place sur tous les éléments autre que le "cd-filter-list". Sa valeur est le nom_du_filtre auquel vous souhaitez vous lier.

‍

groupe de filtres

Je veux que cet élément serve de filtre pour la liste à filtrer.
Ajouter l'attribut
name
cd-filter-group
value
nom_du_groupe
Description

Créé un groupe qui va permettre de filtrer les éléments. C'est ce qui va donner vie au filtre

Le nom_du_groupe doit être unique, et va servir à identifier le groupe dans des évènements ou d'autres attributs.

Pour aller plus loin

Type du groupe de filtres

Je veux définir le type de filtre (bouton, checkbox, select, etc)
Ajouter l'attribut
name
cd-filter-type
value
type_de-groupe
Description

Cet attribut se place à côté du cd-filter-group.

Les valeurs possibles sont :

  • button : l'élément devient un bouton unique permettant de filter la liste
  • buttons : tous les enfants directes du groupe deviennent des boutons individuels (permet d'éviter de mettre le type sur chaque élément individuellement)
  • text : tous les éléments de type "text input" présents dans le groupe deviennent des barres de recherche
  • checkbox : tous les éléments de type checkbox présents dans le groupe deviennent des filtres
  • select : tous les éléments de type "select" présents dans le groupe deviennent des filtres
  • auto : valeur par défaut qui permettra au filtre de détecter automatiquement le type à utiliser entre text, checkbox ou select

Combien de temps dure le développement ?

Le temps de développement dépend du projet que vous souhaitez mener (taille du site, fonctionnalités avancées ou non, application, etc), et des ressources que vous êtes en mesure de fournir (maquettes, images, logo, charte graphique, etc) ✨

‍

‍

  • Projet de site internet avec maquettes déjà faite par un designer, animations d'arrivées simples : environ 15j de développement
  • Projet de site internet avec création des designs directement sur Webflow : environ 1 mois de design et développement
  • Application simple (compte client et quelques fonctionnalités) avec maquettes déjà faites par un designer : environ 2 mois de développement

‍
